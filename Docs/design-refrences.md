> **Purpose:**
> This document defines the exact design language, interaction style, and visual references that the AI must follow.

>
> Instead, each reference contributes one specific design principle.
>
> **Read After:** `PRD.md`

# Design Goal

The portfolio should feel like a **premium software product**, not a resume website.

A recruiter should immediately think:

> "This person has product design taste and engineering skills."

Overall style:

* Minimal
* Premium
* Modern
* Editorial
* Data-focused
* Interactive
* Clean
* High-end

# Things to Avoid

Never use

* Bootstrap templates
* Generic portfolio layouts
* Random gradients
* Neon effects
* Glassmorphism everywhere
* Overly rounded UI
* Excessive blur
* Inconsistent shadows
* Multiple accent colors
* Cluttered hero section
* Long paragraphs
* Low-quality images
* Placeholder graphics

---

# AI Rules

* Use these references for **style**, not for copying layouts.
* Maintain a consistent visual language across the entire site.
* Every section should feel like part of one cohesive product.
* Prioritize typography, spacing, and motion over decorative effects.
* Keep interactions subtle and purposeful.
* If two design choices conflict, choose the simpler, more readable option.
* Always build a portfolio that feels like a premium SaaS product rather than a traditional resume website.
